SweetSyntax manual

What is SweetSyntax ?

SweetSyntax is an extension for the Unity3D engine that aims to sweeten the syntax of UnityScript and C# mostly by providing syntactic sugars.

SweetSyntax allows you to use two pseudo language : SweetScript and Sweet# that are overlays of UnityScript and C#, respectively.

More precisely, SweetSyntax aims to provide a syntax :

  • more human-friendly, using actual words instead of expressions (ie : "and" instead of "&&").
  • more lightweight, and less verbose. It allows to omit unnecessary code, or provide shortcuts for character-heavy expression (ie : a readonly property in SweetScript : prop PropertyName get;).
  • less error-prone. It allows beginners (and more experienced coder as well !) to focus on the logic of the code more than on syntactic bug-generating details.

Current version : 0.1.0 "beta", 14 July 2012.

How does it works ?

You write your code in .sweetscript and/or .sweetsharp scripts instead of .js and .cs scripts, respectively.
Those scripts will be converted to .js and .cs by the extension whenever one is created or modified.

SweetScript and Sweet# are not completely new languages, they are UntyScript and C#, plus some optionnal sugars. Only those sweet expressions will be converted to their regular equivalent if they are found in the scripts, so you can always use the regular UntyScript and C# syntax within our SweetScript and Sweet# scripts.

How to set it up ?

Downloaded a SweetSyntax.unitypackage. Import it in your project if it is not already done.

Just put the "SweetSyntax" folder anywhere in your project's Assets folder.

Open the extension by clicking "SweetSyntax" in the "Window" menu. As you must keep the extension running at all times, pin the tab somewhere.
